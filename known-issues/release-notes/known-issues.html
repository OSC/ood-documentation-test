Slurm 18 before OOD 1.4¶

Slurm 18 introduced a bug in squeue when attempting to output the field gres or tres. OnDemand versions greater than 1.4 are patched for this bug. Prior versions needed the following initializer to be added to Active Jobs, Dashboard and Job composer in order for it to work properly:

require 'ood_core/job/adapters/slurm'
class OodCore::Job::Adapters::Slurm::Batch
  alias_method :orig_fields, :fields
  def fields
    orig_fields.tap { |f| f.delete(:gres) }
  end
end

File Explorer breaks with odd file names¶

To a greater or less degree XML/HTML strings are valid path names, however the File Explorer does not handle them gracefully; they can break the frontend completely.

Consider the following file paths:

  • <a href='https://www.google.com' />
  • <script>window.alert('hacked');</script>
  • <script>window.alert("hacked");</script>
mkdir -p "<a href='https://www.google.com' />"
tree
.
└── <a href='https:
    └── www.google.com'
        └── >

Note

There is no security vulnerability as a result of this. A user attempting malicious behavior by crafting edge-case file and directory names will only inconvenience themselves.

The naive fix is to simply reject these file names at the frontend. An example of this fix is available on Github at https://github.com/OSC/ood-fileexplorer/pull/199.

A future replacement of the File Explorer will handle these edge cases correctly.
